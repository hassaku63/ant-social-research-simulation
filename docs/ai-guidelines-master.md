# AIアシスタント向けガイドライン（マスター文書）

このファイルは、すべてのAIサービス向けガイドラインの**単一の正規ソース（Single Source of Truth）**です。

## ⚠️ 更新ルール（必ず遵守）

このファイルを更新した場合は、**以下のすべてのファイルを同時に更新**すること。
内容の不整合・漏れは、AIアシスタントの動作に一貫性のない結果をもたらすため厳禁。

| 対象ファイル | 対象AIシステム |
|------------|--------------|
| `.github/copilot-instructions.md` | GitHub Copilot |
| `CLAUDE.md` | Claude Code |
| `AGENTS.md` | OpenAI Codex / 汎用AIエージェント |
| `.kiro/steering/project.md` | Amazon Kiro |

各ファイルはAIシステム固有のフォーマット・慣例に合わせた記述になっているが、
**内容（リポジトリの目的・研究テーマ・タスクサイクル・ディレクトリ規約）は本ファイルと一致**させること。

---

## リポジトリの目的

社会性昆虫（蟻・蜂）の集団知性（Swarm Intelligence）を研究・応用するリポジトリ。
個体としては単純なルールに従いながら、集団として高効率な行動を実現するメカニズムを解明し、
人間社会・ソフトウェアへの応用を探ることが目的。

---

## 2つの研究テーマ

### テーマ1: 先行研究調査

蟻・蜂の行動システムに関する先行研究を収集・整理し、応用事例を調査してドキュメント化する。

- 行動システムの仕組み（アルゴリズム・ルール・フィードバック機構）を明らかにする
- 人間社会やソフトウェアへの応用事例を調査する
- なぜ・いかに機能するかを分析する

**成果物保存先**: `docs/research/`

### テーマ2: シミュレーション・応用実装

蟻・蜂の行動システムを応用できる実問題を検討し、設計・実装・検証を反復する。

- コンセプト検討→計画→仕様策定→実装→検証のサイクルを繰り返す
- 応用の有効性・限界・課題を明らかにする

**仕様保存先**: `docs/simulation/` / **コード保存先**: `src/`

---

## タスクサイクル（必ず遵守）

すべての作業は以下のサイクルを反復する:

```
計画の言語化 → 詳細化（仕様検討）→ 実行（実装）→ 検証 → 振り返りを記録
```

| ステップ | 内容 | 成果物 |
|---------|------|--------|
| 計画の言語化 | 目標・スコープ・成果物を明文化 | タスク計画ドキュメント |
| 詳細化 | 手順・仕様・評価基準を具体化 | 仕様ドキュメント |
| 実行 | 計画に従い実装・文書作成 | コード / ドキュメント |
| 検証 | 仕様・評価基準との照合 | 検証レポート |
| 振り返り | 学び・課題・次のアクションを記録 | 振り返りメモ |

詳細: [`docs/workflow/task-cycle.md`](workflow/task-cycle.md)

## コーディングスタイル

- コメントと変数名は日本語を優先する（技術的な固有名詞・ライブラリ名は英語のまま可）
- 新規ファイルを作成する前に既存のドキュメントを確認し、重複を避ける
- 実装コードには対応するシミュレーション仕様ドキュメント (`docs/simulation/`) への参照を含める

---

## 作業ガイドライン

- 新しいタスクを開始する前に `docs/masterplan.md` のステータスを確認・更新する
- 新しいタスクを開始する前に `docs/workflow/task-cycle.md` のテンプレートを参照し、計画ドキュメントを作成する
- 実装コードには対応する仕様ドキュメント (`docs/simulation/`) へのリンクを含める
- 振り返り内容は各タスクのドキュメントに追記し、蓄積していく
- 既存ドキュメントと重複する調査・実装は避け、差分・拡張を明示する
