# 調査まとめ：蟻・蜂の行動システムと群知能

## はじめに

本ドキュメントは、蟻・蜂の行動システムの先行研究と、それを人間社会・ソフトウェアに応用した事例の調査成果をまとめたものである。

---

## 1. 生物学的基盤：なぜ群れは賢いのか

### 1.1 創発（Emergence）

個々の行動者が持つ単純なルールの相互作用から、誰も設計していない複雑な集合的挙動が生まれる現象を「創発」と呼ぶ。

> **蟻の例:** 1匹の蟻はただランダムに歩き回り、食料を見つけたらフェロモンを置くだけ。しかしコロニー全体では、数週間もかからず最短経路を発見し、環境の変化（障害物の出現）に適応する。

### 1.2 正のフィードバックと負のフィードバック

群知能システムは常に「正のフィードバック（良い選択肢の強化）」と「負のフィードバック（悪い選択肢の抑制）」のバランスで機能する。

| フィードバック | 蟻の例 | 蜂の例 |
|---------------|--------|--------|
| **正** | 良い経路のフェロモン濃度上昇 | 良い食料源への活発なダンス |
| **負** | フェロモンの自然蒸発 | 停止信号（piping）による不良候補の抑制 |

このバランスがなければ、正のフィードバックだけでは一点に集中してしまい（収束が速すぎる）、負のフィードバックだけでは情報が失われてしまう（探索が無駄になる）。

### 1.3 探索と活用のトレードオフ（Exploration vs. Exploitation）

群知能の核心的な問題の一つが「探索（新しい解を探す）」と「活用（既知の良い解を活用する）」のバランスである。

- **蟻のACO:** フェロモンの揮発率（α）と新規探索の確率（β）がこのバランスを調整する
- **蜂のBCO:** スカウト蜂の割合とダンスの閾値がバランスを制御する

## 2. 応用の成功要因分析

### 2.1 問題の特性との適合性

群知能アルゴリズムが特に有効な問題の特性：

| 問題の特性 | 理由 |
|-----------|------|
| **組み合わせ爆発** | 並列探索により広い解空間を効率的にカバー |
| **動的変化** | フェロモンの蒸発/更新により最新の状況を反映 |
| **NP困難問題** | ヒューリスティックとして近似最適解を現実的時間で発見 |
| **分散環境** | 中央制御なしに機能するため、ネットワーク障害に強い |

### 2.2 ACO の数学的モデル

**フェロモン更新式（ASアルゴリズム）:**

```
τ_ij(t+1) = (1-ρ) * τ_ij(t) + Σ Δτ_ij^k
```

- `τ_ij`: 辺(i,j)のフェロモン濃度
- `ρ`: フェロモン蒸発率（0 < ρ < 1）
- `Δτ_ij^k`: 蟻kが辺(i,j)を通ったときのフェロモン付加量

**移動確率:**

```
p_ij^k = [τ_ij]^α * [η_ij]^β / Σ [τ_il]^α * [η_il]^β
```

- `η_ij`: 辺(i,j)のヒューリスティック情報（例：距離の逆数）
- `α`: フェロモンの重み
- `β`: ヒューリスティック情報の重み

### 2.3 代表的アルゴリズムの比較

| アルゴリズム | 着想 | 主な特徴 |
|-------------|------|--------|
| AS（Ant System） | 蟻フェロモン | 基本的なACOモデル |
| ACS（Ant Colony System） | 蟻フェロモン+局所探索 | より高い収束性能 |
| MMAS（Max-Min AS） | フェロモン上下限制約 | 停滞回避に効果的 |
| ABC（Artificial Bee Colony） | 蜂の採餌 | 数値最適化に優れる |
| BCO（Bee Colony Optimization） | 蜂の意思決定 | 組み合わせ問題向き |

## 4. 課題と限界

### 4.1 収束の問題

- フェロモンが特定の解に偏ると、他の解の探索が不十分になる（早期収束）
- 対策：フェロモン上下限の設定（MMAS）、ランダム探索の保持

### 4.2 パラメータ感度

- α、β、ρ などのパラメータが性能に大きく影響する
- 問題ごとに適切なパラメータ設定が必要で、汎化が難しい

### 4.3 計算複雑性

- 大規模問題では計算時間が増大する
- 並列実装や近似手法との組み合わせが必要

### 4.4 解の品質保証がない

- ヒューリスティックであるため、最適解の保証はない
- 品質要件が厳しい問題には他の手法との組み合わせが必要

## 5. 群知能の本質的な洞察

### 5.1 「愚かな個体」による「賢い集団」

群知能の最大の洞察は、**知性は個体にあるのではなく、個体間の相互作用のパターンにある**ということである。

蟻一匹は脳の大きさからして、複雑な最適化を行う能力はない。しかし、コロニー全体として見ると、巡回セールスマン問題の近似解を自律的に発見できる。

### 5.2 ローカルとグローバルの関係

各個体は**局所的な情報（フェロモン濃度、近隣の状況）**のみに基づいて行動するが、全体として**グローバルな最適化**が達成される。

これは分散システム設計の重要な教訓を与える：
- 各コンポーネントは局所的な情報だけで動作できるように設計する
- グローバルな調整は、局所的相互作用の集積として創発させる

### 5.3 エラーの活用

群知能システムでは「完全な個体」は必要なく、むしろ**確率的なエラー（ばらつき）**が探索の多様性を保つ上で重要な役割を果たす。
