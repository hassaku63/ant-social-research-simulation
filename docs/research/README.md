# テーマ1: 先行研究調査 ガイドライン

このドキュメントは **必要時参照** 用のリファレンスです。
研究・調査タスクを開始する際に参照してください。

## このテーマの目的

蟻・蜂などの社会性昆虫の行動システムに関する先行研究を収集・整理し、以下を明らかにします:

1. 蟻・蜂の行動システムの仕組み（アルゴリズム・ルール・フィードバック機構）
2. それが人間社会やソフトウェアに応用された事例
3. なぜ・いかに機能するかのメカニズム

## 調査対象トピック（例）

### 行動システム
- フェロモンを用いた経路探索（Ant Colony Optimization: ACO）
- タスク分担・役割分化（労働分業）
- 群知能による集団意思決定
- スティグマジー（環境を媒介とした間接的コミュニケーション）
- 蜂のワッグルダンスによる情報伝達

### 応用事例
- ルーティング・最適化アルゴリズム（TSP、ネットワークルーティング）
- ロボット群（Swarm Robotics）
- 組織管理・プロジェクト管理への応用
- ソフトウェアアーキテクチャへの応用

## ディレクトリ構成

```
docs/research/
├── README.md               # このファイル（調査ガイドライン）
├── index.md                # 調査済みトピックの一覧・索引
├── topics/                 # トピック別の調査ドキュメント
│   ├── aco/                # Ant Colony Optimization
│   ├── swarm-robotics/     # スウォームロボティクス
│   └── ...
└── references.md           # 参考文献リスト
```

## 調査ドキュメントの書き方

各調査ドキュメントは次の構成で作成します:

```markdown
# [トピック名]

## 概要
（2〜3文でトピックの説明）

## メカニズム
（どのように機能するか）

## 重要な研究・文献
（著者、年、タイトル、URL/DOI）

## 応用事例
（実際に応用された事例と効果）

## なぜ機能するか（考察）
（このシステムが有効な理由の考察）

## 関連トピック
（関連する他の調査ドキュメントへのリンク）

## 振り返り
（調査を通じて得た学び・残った疑問）
```

## 品質基準

調査ドキュメントは以下の基準を満たすこと:

- [ ] 出典（著者・年・タイトル・URL/DOI）が明記されている
- [ ] 一次資料（論文・学術書）を少なくとも1件参照している
- [ ] メカニズムが具体的に説明されている（「なんとなく」ではない）
- [ ] 応用事例が1件以上記載されている
- [ ] 関連トピックへのリンクが含まれている
- [ ] 振り返りが記録されている

## 新しい調査を始めるときの手順

1. `docs/research/index.md` を確認し、既存の調査と重複しないか確認する
2. `docs/workflow/templates/research-task.md` を使って計画を立てる
3. 調査ドキュメントを `docs/research/topics/<topic-name>/` に作成する
4. 完了後は `docs/research/index.md` と `docs/research/references.md` を更新する
