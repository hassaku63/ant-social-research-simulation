# 蜂のワッグルダンス（Waggle Dance）深掘り調査

> 本ドキュメントは既存の概要調査（[蜂の行動システム](../../02_bee_behavior_systems.md)）を補完する深掘り資料です。
> ワッグルダンスの信号構造・情報符号化の精度・集団意思決定（Honeybee Democracy）・
> アルゴリズムや人間社会への応用を掘り下げます。

## 概要

ワッグルダンス（Waggle Dance）は、ミツバチ（*Apis mellifera*）が食料源や巣の候補地の位置・品質を仲間に伝達するために行う身体的コミュニケーションである。Karl von Frisch が1940年代に解読し、1967年の著書で体系化、1973年にノーベル生理学・医学賞を受賞した (von Frisch, 1967)。

ワッグルダンスは、方向・距離・品質という3次元情報を身体運動に符号化する高度な情報伝達システムであり、社会性昆虫の直接的コミュニケーションの代表例である。スティグマジー（環境を介した間接的コミュニケーション）とは対照的なメカニズムであり、両者の比較は集団知性の理解に重要な視座を提供する。

## 1. ワッグルダンスの信号構造

### 1.1 ダンスの物理的構成

ワッグルダンスは「8の字」の動きで構成される:

```
       ╱ ─ ╲
      │     │  ← 右回りの円弧（リターンフェーズ）
       ╲   ╱
        │↑│    ← 直進部分（ワッグルラン）: 腹部を左右に振りながら直進
       ╱   ╲
      │     │  ← 左回りの円弧（リターンフェーズ）
       ╲ ─ ╱
```

1. **ワッグルラン（直進部分）**: 腹部を左右に振りながら直進する。情報の核心
2. **リターンフェーズ**: 右回り→左回りを交互に行い、開始位置に戻る

### 1.2 符号化される3つの情報

| 情報 | 符号化方法 | 精度 |
|------|-----------|------|
| **方向** | ワッグルランの角度（重力方向に対する角度 = 太陽方向に対する食料源の角度） | ±10〜15度 |
| **距離** | ワッグルランの持続時間（長いほど遠い） | 種により異なるが概ね正確 |
| **品質** | ダンスの活発さ・繰り返し回数（高品質ほど長く激しく踊る） | 相対的な比較に有効 |

### 1.3 方向の符号化: 太陽コンパス

ミツバチは巣の中の暗い垂直面でダンスを行う。太陽の方向を重力方向（上）に置き換えて表現する:

```
巣の外の世界（水平面）          巣の中（垂直面）

      太陽                         上（= 太陽方向）
       ↑                            ↑
       |  θ°                         |  θ°
  巣 ──┤─── → 食料源          ──────┤──── → ワッグルランの方向
```

- ワッグルランが真上に向かう → 食料源は太陽の方向
- ワッグルランが右に30度傾く → 食料源は太陽から右に30度の方向
- ワッグルランが真下 → 食料源は太陽の反対方向

**注目すべき点**: ミツバチは時間の経過に伴う太陽の移動を補正する。長時間ダンスを続ける場合、ワッグルランの角度を太陽の移動分だけ調整する (von Frisch, 1967)。

### 1.4 距離の符号化: 持続時間

ワッグルランの持続時間と距離の関係は種によって異なるが、概ね線形:

| ワッグルラン持続時間 | おおよその距離 |
|-------------------|-------------|
| 約0.5秒 | 500m |
| 約1.0秒 | 1,000m |
| 約2.0秒 | 2,000m |

距離が短い場合（約50m以内）は、方向情報を含まない「円形ダンス（round dance）」を行う。これは「近くに食料源がある」という情報のみを伝える (von Frisch, 1967)。

### 1.5 品質の符号化: ダンスの活発さ

- ダンスの繰り返し回数（サーキット数）が品質に比例する
- 高糖度の花蜜を見つけた蜂はより長く・激しくダンスを続ける
- 品質が低い場合、蜂はすぐにダンスを止めるか、そもそもダンスしない

この仕組みにより、コロニー全体として**高品質な食料源に多くのフォロワーが向かう**という資源配分の最適化が自律的に実現される。

## 2. 情報伝達の精度と限界

### 2.1 方向の誤差

von Frisch (1967) および後続の研究により、以下が明らかになっている:

- ワッグルランの方向の標準偏差は約 ±10〜15度
- この誤差は食料源がコロニーに近いほど大きくなる（近距離では方向情報の価値が低いため、進化的に精度が低くても問題ない）
- 複数のダンスを観察することで、フォロワー蜂は方向の推定精度を上げる

### 2.2 ノイズの機能的役割

Dornhaus & Chittka (2004) は、ワッグルダンスの方向誤差がむしろ有利に働く場合があることを示した:

- 方向の「ノイズ」により、フォロワー蜂は報告された地点の周辺を広く探索する
- 食料源がパッチ状に分布する場合、この「拡散効果」がより多くの花を発見させる
- **探索と活用のバランス**: 正確すぎる情報は活用に偏り、適度なノイズが探索を促す

これはACOにおける確率的遷移と類似の原理であり、群知能に共通する「最適なノイズ量」の問題である。

### 2.3 ダンスを読むプロセス

フォロワー蜂がダンスから情報を得るプロセス:

1. ダンサーの近くに寄り、触角でダンサーの体を追跡する
2. 複数回のワッグルランを追跡して平均的な方向・距離を推定する
3. ダンサーが運んできた花蜜のサンプルを受け取り、食料源の種類を特定する（嗅覚情報）
4. 巣を飛び立ち、推定した方向・距離で食料源を探索する

**重要**: フォロワー蜂は通常1匹のダンサーのダンスのみを追跡する。複数のダンサーを比較して最良を選ぶわけではない（採餌の場合）。集団レベルでの最適化は、個々のフォロワーの行動の集積として創発する。

## 3. Honeybee Democracy: 集団意思決定

Thomas Seeley の研究 (Seeley, 2010) により、分蜂時の巣場所選択プロセスが詳細に解明された。

### 3.1 分蜂のコンテキスト

分蜂（swarming）: 女王蜂が約半数の働き蜂を連れて古い巣を離れ、新しい巣を探すプロセス。コロニーの存続に関わる重大な意思決定である。

### 3.2 意思決定プロセスの詳細

```
フェーズ1: 探索
  スカウト蜂（全体の約3〜5%）が候補地を探索
  ↓
フェーズ2: 評価と広告
  各スカウトが候補地の品質を評価し、ワッグルダンスで広告
  高品質な場所ほど長く激しくダンス
  ↓
フェーズ3: 議論と合意形成
  スカウト蜂が他のスカウトのダンスを見て候補地を訪問
  より良い場所を発見するとダンスを乗り換える
  ↓
フェーズ4: 定足数（クオラム）検出
  一つの候補地にスカウトが一定数（約20〜30匹）集まると定足数達成
  ↓
フェーズ5: 移動
  全蜂が一斉に新しい巣へ移動
```

### 3.3 意思決定を支える5つのメカニズム

Seeley (2010) は、蜂の意思決定を民主的プロセスとして分析し、以下のメカニズムを特定した:

| # | メカニズム | 機能 |
|---|----------|------|
| 1 | **多様な選択肢の生成** | 数百匹のスカウトが独立に候補地を探索し、多様な選択肢を発見 |
| 2 | **自由で開かれた競争** | 各スカウトが自由にダンスし、他のスカウトのダンスも自由に観察できる |
| 3 | **正確な品質評価** | スカウトは候補地を直接訪問し、複数の基準で評価する |
| 4 | **ポジティブフィードバック** | 高品質な候補地ほど多くのダンスを引きつけ、支持が加速する |
| 5 | **停止信号（ネガティブフィードバック）** | 競合する候補地を支持するスカウトに対して「停止信号（stop signal）」を送り、ダンスを抑制する |

### 3.4 停止信号の役割

Seeley et al. (2012) は停止信号の重要性を実験的に示した:

- 蜂が別の蜂の体に頭突きをしながら短い振動を与える行動
- 競合する候補地のダンサーに対して行われる
- 効果: ダンスの継続時間を短くし、劣った候補地への支持を減速させる
- **機能**: 正のフィードバックだけでは複数の候補地が拮抗する「デッドロック」に陥りうるが、停止信号がこれを打破する

### 3.5 定足数検出（クオラムセンシング）

合意の判定にはクオラム（定足数）メカニズムが使われる:

- スカウト蜂が候補地を訪問した際、そこにいるスカウトの数を感知する
- 一定数（約20〜30匹）を超えると「定足数に達した」と判断し、帰巣してパイピング信号（翅の振動音）を発する
- パイピング信号が群れ全体に伝播し、飛翔筋の温度を上げて移動の準備をする

**注目すべき点**: 定足数による意思決定は、細菌のクオラムセンシング（一定密度で遺伝子発現が変化する）と原理的に類似しており、生物界に広く見られるパターンである。

## 4. ワッグルダンスとスティグマジーの比較

ワッグルダンスとスティグマジーは、社会性昆虫における2つの対照的なコミュニケーション戦略である。

| 特性 | ワッグルダンス | スティグマジー |
|------|-------------|-------------|
| コミュニケーション方式 | **直接的**: 個体間の対面コミュニケーション | **間接的**: 環境を介したコミュニケーション |
| 情報の媒体 | 身体運動（ダンス） | 環境の変化（フェロモン、構造物） |
| 情報の持続性 | 一時的（ダンス中のみ） | 持続的（蒸発・風化まで） |
| 情報の複雑さ | 高い（方向・距離・品質を同時に伝達） | 低い（濃度や状態のみ） |
| スケーラビリティ | 制限あり（ダンスを見れる蜂の数に依存） | 高い（環境を介するため個体数に依存しにくい） |
| 同期性 | 同期的（ダンサーとフォロワーが同時に存在する必要） | 非同期的（時間差があっても機能） |

**両者の相補性**: 蜂はワッグルダンス（直接的・高情報量）とフェロモン（間接的・低情報量）の両方を使い分ける。スカウト蜂がダンスで高精度な位置情報を伝え、実際の食料源ではフェロモンマーキングで仲間を誘導する。

## 5. アルゴリズムへの応用

### 5.1 Bee Algorithm (BA)

**提案**: Pham et al. (2005)

ワッグルダンスの情報共有メカニズムを最適化アルゴリズムに応用:

| アルゴリズム要素 | ワッグルダンスとの対応 |
|---------------|---------------------|
| スカウト蜂 | 解空間のランダム探索 |
| エリートサイト | 高品質な食料源（良い解の近傍） |
| フォロワーの割り当て | ダンスの激しさに比例（良い解に多くの探索資源を配分） |
| 近傍探索 | 食料源周辺の探索 |

### 5.2 Artificial Bee Colony (ABC)

**提案**: Karaboga (2005)

3種類の蜂の役割分担を模倣:

| 役割 | アルゴリズムでの機能 |
|------|-------------------|
| エンプロイドビー（Employed Bee） | 既知の解の近傍を探索し改善する |
| オンルッカービー（Onlooker Bee） | 解の品質に比例した確率で解を選択し、近傍探索する |
| スカウトビー（Scout Bee） | 長期間改善されない解を放棄し、新しいランダム解を生成する |

オンルッカービーの選択確率がワッグルダンスに対応する: 品質の高い解（活発なダンス）により多くのオンルッカーが集まる。

### 5.3 ACOとの比較

| 特性 | ACO | Bee Algorithm / ABC |
|------|-----|---------------------|
| 生物学的基盤 | フェロモントレイル（スティグマジー） | ワッグルダンス（直接通信） |
| 情報共有 | 環境に蓄積（フェロモン行列） | 個体間の直接伝達（適応度比例選択） |
| 探索戦略 | 構成的（解を逐次構築） | 摂動的（既存解を改良） |
| 得意な問題 | 組み合わせ最適化（TSP, VRP等） | 連続値最適化（関数最適化等） |

## 6. 人間社会への応用と示唆

### 6.1 Honeybee Democracy と人間の意思決定

Seeley (2010) は蜂の意思決定プロセスから人間の組織に対する教訓を導出した:

| 蜂のメカニズム | 人間の組織への示唆 |
|-------------|-----------------|
| 多様なスカウトによる探索 | 多様な視点を持つメンバーで選択肢を広く探索する |
| 自由で開かれた議論 | 権威に抑圧されない自由な議論を保証する |
| 品質に基づく直接評価 | 伝聞ではなく、各自が選択肢を直接評価する |
| 停止信号 | 劣った選択肢を明示的に否定するメカニズムを持つ |
| 定足数による決定 | 全員一致ではなく、十分な支持が集まった時点で決定する |

### 6.2 陪審制度との類似性

List (2004) は蜂の集団意思決定と陪審制度の類似性を分析:

- 両者とも独立した個体が証拠を評価し、集団として判断する
- コンドルセの陪審定理: 各個体が50%以上の確率で正しい判断をする場合、集団の判断精度は個体数の増加とともに1に近づく
- 蜂の場合、スカウトの品質評価は個体レベルでも高精度（Seeley & Buhrman, 2001）であり、集団レベルでの正確さはさらに高い

### 6.3 分散型チーム運営への適用

ワッグルダンスモデルから導かれる分散型チーム運営の原則:

1. **情報の透明性**: チームメンバー全員が選択肢の情報にアクセスできること（ダンスの公開性）
2. **品質ベースの資源配分**: 有望なプロジェクトに自然と人的リソースが集まる仕組み（ダンスの活発さ ∝ 品質）
3. **撤退の仕組み**: 見込みのないプロジェクトから自然に人が離れる仕組み（ダンスの停止）
4. **十分条件での意思決定**: 完璧な合意を待たず、十分な支持で前に進む（クオラム）

## なぜ機能するか（考察）

ワッグルダンスによる集団意思決定が機能する本質的な理由:

1. **情報の分散収集**: 数百匹のスカウトが並列に候補を探索し、単一個体では不可能な広域探索を実現
2. **品質比例の増幅**: ダンスの活発さが品質に比例することで、正のフィードバックが良い選択肢を増幅する
3. **停止信号による競合解消**: 負のフィードバックがデッドロックを防ぎ、合意への収束を加速する
4. **クオラムによる精度と速度の両立**: 全員一致を求めず定足数で決定することで、迅速かつ十分に正確な意思決定を実現
5. **独立した評価**: 各スカウトが候補地を直接評価するため、情報のカスケード（伝聞による歪み）が軽減される
