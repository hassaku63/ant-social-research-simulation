# スティグマジー（Stigmergy）深掘り調査

> 本ドキュメントは既存の概要調査（[蟻の行動システム](../../01_ant_behavior_systems.md)）を補完する深掘り資料です。
> スティグマジーの理論的基盤・分類・自然界の事例・ACOとの接続・人間社会/ソフトウェアへの応用を掘り下げます。

## 概要

スティグマジー（Stigmergy）は、個体が環境を変化させ、その変化が他の個体の行動を誘発する間接的コミュニケーション機構である。1959年にフランスの昆虫学者 Pierre-Paul Grasse がシロアリの巣構築行動を説明するために提唱した概念であり、社会性昆虫の集団知性の根幹をなすメカニズムである (Grasse, 1959)。

中央制御なしに複雑な協調行動が実現される仕組みとして、生物学にとどまらず、コンピュータサイエンス・ロボティクス・組織論・Web技術に至る幅広い分野で応用されている。

---

## 1. 定義と語源

### 1.1 語源

ギリシャ語の2つの単語に由来する:

- **stigma**（στίγμα）: 刺す、印をつける
- **ergon**（ἔργον）: 仕事、作業

すなわち「作業の痕跡が次の作業を誘発する」という意味を持つ。

### 1.2 定義

Grasse (1959) の原義:

> 個体の作業の結果として生じた環境の変化が、同じ個体または他の個体の次の作業を刺激・誘導する現象

より一般的に定式化すると:

```
個体 A が環境を変化させる → 環境の変化が残る → 個体 B がその変化を感知 → 個体 B の行動が変化
```

**重要な特徴**: 個体間の直接的なコミュニケーション（対面、信号の送受信等）を必要としない。環境自体が「共有メモリ」として機能する。

## 2. スティグマジーの分類

Theraulaz & Bonabeau (1999) による分類:

### 2.1 定量的スティグマジー（Quantitative Stigmergy）

環境に残された**量的な情報**（濃度、強さ等）が行動を誘導する。

**特徴**:
- 刺激の強度が行動の強度・確率に比例する
- 正のフィードバックを生みやすい
- 時間とともに減衰する（蒸発・拡散）

**代表例**: 蟻のフェロモントレイル
- フェロモン濃度が高いほど、その経路を選択する確率が上がる
- 濃度は連続的な量であり、蓄積・蒸発する

### 2.2 定性的スティグマジー（Qualitative Stigmergy）

環境に残された**構造的・質的な情報**が、次に行うべき作業の種類を決定する。

**特徴**:
- 作業の現在状態が次の作業の種類を決める
- 段階的なプロセスを制御する
- 構造物の形状・状態がトリガーとなる

**代表例**: シロアリの巣構築
- 土の柱がある高さに達すると、アーチの構築が開始される
- 柱の状態（高さ・形状）が次の建築行動を質的に決定する

### 2.3 比較

| 特性 | 定量的 | 定性的 |
|------|--------|--------|
| 情報の性質 | 濃度・強度（連続量） | 形状・状態（離散的カテゴリ） |
| フィードバック | 正のフィードバックが支配的 | 段階的な状態遷移 |
| 減衰 | 時間とともに蒸発・拡散 | 構造として残る（永続的） |
| 代表例 | フェロモントレイル | 巣の構築 |
| ACOとの関係 | フェロモン値 τ_ij に直接対応 | 該当なし |

## 3. 自然界におけるスティグマジー

### 3.1 蟻のフェロモントレイル（定量的）

最も研究されたスティグマジーの事例。

- 蟻が経路上にフェロモンを分泌し、他の蟻がそれを感知して経路選択する
- 短い経路ほどフェロモンが蓄積しやすく、最短経路が自動的に強化される
- 複数種のフェロモンを使い分ける（道しるべ、アラーム、集合等）

> 詳細は [蟻の行動システム概要](../../01_ant_behavior_systems.md) および [ACO深掘り調査](../aco/) を参照。

### 3.2 シロアリの巣構築（定性的）

Grasse (1959) が最初にスティグマジーを記述した対象。

**巣構築のプロセス**:

1. シロアリが唾液で土の粒を固め、ランダムな場所に積む
2. 積まれた土の粒にはフェロモンが含まれ、近くのシロアリを誘引する（定量的要素）
3. 土の柱がある高さに達すると、シロアリは横方向に建築を開始する（定性的要素）
4. 隣り合う柱がアーチとして結合され、複雑な巣構造が形成される

**注目すべき点**: シロアリの脳は極めて小さく、巣全体の設計図を持つことは不可能。環境に残された建築の痕跡のみから、巨大で精巧な構造物が創発的に生まれる。

### 3.3 蜂の巣房構築（定性的）

ミツバチの六角形の巣房も定性的スティグマジーの例:

- 蜂は隣接する巣房の壁の状態を感知し、次の巣房を適切な位置に構築する
- 六角形のパッキングが最も材料効率の良い構造であり、局所的なルールから最適な幾何構造が創発する

### 3.4 蟻の死体処理（定量的 + 定性的）

蟻のコロニーでは死体を「墓地」に集積する:

- 死体から放出される化学物質（オレイン酸等）を感知する（定量的）
- 死体が既に集まっている場所に新たな死体を置く（正のフィードバック）
- 結果として、コロニー内に自発的に「墓地クラスタ」が形成される

## 4. スティグマジーとACOの理論的接続

ACOはスティグマジーをアルゴリズムとして形式化した最も成功した例である。

### 4.1 対応関係

| スティグマジーの要素 | ACOにおける実装 |
|-------------------|---------------|
| 環境への痕跡 | エッジ上のフェロモン値 τ_ij |
| 痕跡による行動誘導 | 遷移確率 p_ij ∝ τ_ij^α |
| 正のフィードバック | 良い解のエッジにフェロモンを追加 |
| 負のフィードバック（減衰） | フェロモン蒸発 τ_ij ← (1-ρ) * τ_ij |
| 環境が共有メモリ | フェロモン行列が全蟻で共有 |

### 4.2 ACOがスティグマジーを拡張している点

1. **ヒューリスティック情報の追加**: 自然のスティグマジーは環境の痕跡のみだが、ACOでは問題固有の情報（距離の逆数等）を組み合わせる
2. **明示的な評価**: 解全体を評価してからフェロモンを更新する（自然界では逐次的）
3. **パラメータによる制御**: α, β, ρ 等でフィードバックの強さを調整可能

### 4.3 スティグマジーの視点から見たACOバリエーション

| バリエーション | スティグマジーの観点 |
|-------------|-------------------|
| AS | 全蟻が痕跡を残す（自然界に最も近い） |
| ACS | 最良蟻のみが痕跡を残す + 局所更新で痕跡を消す（探索の多様性確保） |
| MMAS | 痕跡の量に上下限を設ける（過度な集中の防止） |

> ACOの各バリエーションの詳細は [ACO深掘り調査](../aco/) を参照。

## 5. ソフトウェア・Webへの応用

スティグマジーの原理は、意図せずして多くのソフトウェアシステムに内在している。

### 5.1 Wiki / Wikipedia

**スティグマジーの実現**:
- ユーザーがページを編集する（環境への痕跡）
- 編集された内容が次の編集者の行動を誘導する（痕跡による行動誘導）
- 良い記事ほど多くの編集を引きつけ、品質が向上する（正のフィードバック）
- 古い情報は他の編集者によって修正・削除される（負のフィードバック）

Elliott (2006) は Wikipedia をスティグマジーの観点から分析し、中央制御なしに大規模な知識ベースが自己組織化される仕組みを論じた。

### 5.2 バージョン管理（Git / GitHub）

- コミット・ブランチ・プルリクエストが環境への痕跡として機能
- Issues や PR のコメントが次のコントリビューションを誘導
- スター数やフォーク数が人気プロジェクトに貢献者を集める（正のフィードバック）

### 5.3 Stack Overflow / Q&A サイト

- 質問が回答を誘導し、回答が新たな質問を生む
- 投票システムが良い回答を強化し、劣った回答を沈める（定量的スティグマジー）
- タグシステムが関連する質問を結びつける

### 5.4 マルチエージェントシステム

人工スティグマジーはマルチエージェントシステムの協調メカニズムとして広く研究されている:

- **デジタルフェロモン**: エージェントが仮想環境にマーカーを配置し、他のエージェントがそれを読み取る
- **共有ブラックボード**: エージェント間の直接通信を避け、共有データ構造を介して協調する
- **適用例**: ロボット群の探索、分散タスク割り当て、ネットワークルーティング

### 5.5 推薦システム

- ユーザーの購買・閲覧履歴が環境への痕跡
- 「この商品を買った人はこれも買っています」は集団的なスティグマジー
- 人気商品ほど露出が増え、さらに購買される（正のフィードバック）

## 6. 人間社会への応用

### 6.1 都市の自己組織化

- 人が多く通る場所に店舗が集まり、店舗が集まることでさらに人が集まる
- 「けもの道」: 人が繰り返し通ることで自然に形成される道（物理的スティグマジー）
- Jane Jacobs (1961) の都市論は、明示的にスティグマジーを参照していないが、同じ原理を記述している

### 6.2 オープンソースコミュニティ

Heylighen (2007) はオープンソース開発をスティグマジーの観点から分析:

- コードそのものが共有環境であり、痕跡（コミット）が次の開発を誘導する
- 中央制御者なしに（メンテナーの役割はあるが）複雑なソフトウェアが構築される
- バグレポートが修正を誘導し、修正が新たな開発を誘導する

### 6.3 組織マネジメント

- カンバンボード: タスクの状態が視覚的な痕跡として機能し、次の作業を誘導する
- 共有ドキュメント: 書かれた内容が次の執筆者の行動を誘導する
- スティグマジーの観点では、優れた組織ツールとは「環境を適切に可視化し、適切な行動を自然に誘導する」ものである

## 7. なぜ機能するか（考察）

スティグマジーが有効に機能するための条件:

### 7.1 必要条件

1. **共有環境の存在**: 全個体がアクセスできる環境（物理空間、データベース、コードベース等）
2. **痕跡の残存性**: 行動の結果が一定期間環境に残ること
3. **感知能力**: 個体が環境の変化を感知し、行動を変えること
4. **適度な減衰**: 痕跡が永続しすぎず、状況の変化に適応できること

### 7.2 スティグマジーが直接通信より優れる場面

| 条件 | スティグマジーの優位性 |
|------|---------------------|
| 個体数が多い | N個体の直接通信は O(N^2) だが、環境経由は O(N) |
| 非同期的な協調 | 個体が同時にいなくても情報が環境に残る |
| 匿名性 | 誰が痕跡を残したかを知る必要がない |
| ロバスト性 | 個体の離脱・加入に対してシステムが頑健 |
| スケーラビリティ | 個体数の増減に対してメカニズムが変わらない |

### 7.3 限界

- **情報の質**: 痕跡は単純な信号であり、複雑な意図の伝達には不向き
- **収束の遅さ**: 直接通信に比べ、集団が良い解に到達するまで時間がかかりうる
- **ノイズへの脆弱性**: 環境ノイズが痕跡を乱すと、誤った行動が誘導される
- **負のフィードバックの設計**: 減衰が弱すぎると停滞、強すぎると学習が失われる
