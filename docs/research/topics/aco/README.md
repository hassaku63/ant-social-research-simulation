# Ant Colony Optimization (ACO) 深掘り調査

> 本ドキュメントは既存の概要調査（[蟻の行動システム](../../01_ant_behavior_systems.md)、[応用事例](../../03_applications.md)）を補完する深掘り資料です。
> ACOの理論的基盤、主要バリエーション、収束性、パラメータ設計を掘り下げます。

## 概要

Ant Colony Optimization (ACO) は、蟻のフェロモンによる経路探索行動を模倣したメタヒューリスティック最適化フレームワークである。Marco Dorigoが1992年の博士論文で提案した Ant System (AS) を起点に、複数のバリエーションが開発され、NP困難な組み合わせ最適化問題に対する有力な解法として確立された (Dorigo & Stutzle, 2004)。

---

## 1. 自然界からアルゴリズムへ: 理論的基盤

### 1.1 自然界の蟻の経路探索

蟻は以下の単純なルールで最短経路を発見する (Deneubourg et al., 1990):

1. 移動中にフェロモンを分泌する
2. フェロモン濃度の高い方向を確率的に選好する
3. フェロモンは時間とともに蒸発する

この3つのルールにより**正のフィードバックループ**が形成される:

```
短い経路 → 往復が速い → フェロモン蓄積が速い → より多くの蟻が選択 → さらにフェロモン蓄積
```

同時にフェロモンの蒸発が**負のフィードバック**として機能し、劣った経路が徐々に忘れられる。

### 1.2 ダブルブリッジ実験

Deneubourg et al. (1990) の実験で、アルゼンチンアリ (*Linepithema humile*) が2本の異なる長さの橋から短い方を選択することを実証した。この実験はACOの生物学的妥当性の根拠となっている。

### 1.3 アルゴリズムへの抽象化

自然界の行動をアルゴリズムに変換する際の対応関係:

| 自然界の要素 | ACOアルゴリズムの構成要素 |
|------------|----------------------|
| 蟻 | 人工蟻（ソリューション構築エージェント） |
| フェロモン | エッジ上のフェロモン値 τ_ij |
| フェロモン蒸発 | 蒸発率 ρ による減衰 |
| 距離の近さの選好 | ヒューリスティック情報 η_ij = 1/d_ij |
| 確率的経路選択 | 遷移確率 p_ij |
| 最短経路の強化 | 良い解のエッジにフェロモンを追加付与 |

自然界と異なり、ACOでは以下の設計上の拡張が加えられている:

- **ヒューリスティック情報の導入**: 蟻は距離情報を直接知らないが、ACOでは距離の逆数を明示的に利用する
- **フェロモン更新の制御**: 全蟻でなく最良解の蟻のみが更新する等のバリエーション
- **解の構築後の評価**: ツアー全体を評価してからフェロモンを更新する（蟻は逐次的に分泌）

## 2. Ant System (AS): 原型アルゴリズム

Dorigo (1992) が提案した最初のACOアルゴリズム。

### 2.1 アルゴリズムの流れ

```
初期化: 全エッジのフェロモンを τ_0 に設定

繰り返し (各イテレーション t):
  1. 解の構築:
     - 各蟻 k がランダムな開始都市から出発
     - 未訪問都市を確率的に選択して巡回ルートを構築
  2. フェロモン更新:
     - 全エッジのフェロモンを蒸発: τ_ij ← (1-ρ) * τ_ij
     - 各蟻 k のルートに沿ってフェロモンを追加: τ_ij ← τ_ij + Δτ_ij^k
```

### 2.2 遷移確率

蟻 k が都市 i から未訪問都市 j を選択する確率:

```
p_ij^k = [τ_ij]^α * [η_ij]^β / Σ_{l ∈ N_k} [τ_il]^α * [η_il]^β
```

- `τ_ij`: エッジ (i,j) のフェロモン濃度
- `η_ij = 1/d_ij`: ヒューリスティック情報（距離の逆数）
- `α`: フェロモンの影響度
- `β`: ヒューリスティック情報の影響度
- `N_k`: 蟻 k の未訪問都市集合

### 2.3 フェロモン更新

```
τ_ij(t+1) = (1-ρ) * τ_ij(t) + Σ_k Δτ_ij^k

Δτ_ij^k = Q / L_k   (蟻 k がエッジ (i,j) を通った場合)
         = 0          (通らなかった場合)
```

- `ρ ∈ (0,1]`: 蒸発率
- `Q`: フェロモン定数
- `L_k`: 蟻 k のツアー総距離

### 2.4 ASの特徴と限界

**強み**:
- アルゴリズムが単純で理解しやすい
- 全蟻がフェロモンを更新するため多様な探索が可能

**限界**:
- 大規模問題（都市数が多い）で収束が遅い
- フェロモンが分散し、良い解への集中が弱い
- パラメータ感度が高い

## 3. 主要バリエーション

ASの限界を克服するために、複数の改良アルゴリズムが提案された。

### 3.1 Ant Colony System (ACS)

**提案**: Dorigo & Gambardella (1997)

ASからの主な改良点:

| 改良点 | AS | ACS |
|-------|-----|-----|
| 遷移ルール | 確率的選択のみ | 擬似ランダム比例ルール（q_0で決定的/確率的を切替） |
| フェロモン更新 | 全蟻が更新 | **最良蟻のみ**が更新（グローバルベスト or イテレーションベスト） |
| 局所フェロモン更新 | なし | 蟻が通過する度にフェロモンを**減少**させる |

**擬似ランダム比例ルール**:

```
q を [0,1] の一様乱数とし、パラメータ q_0 ∈ [0,1] に対して:

q ≤ q_0 の場合: j = argmax_{l ∈ N_k} [τ_il]^α * [η_il]^β  (決定的: 活用)
q > q_0 の場合: ASと同じ確率的選択                             (確率的: 探索)
```

`q_0` が大きいほど「活用（exploitation）」寄り、小さいほど「探索（exploration）」寄りになる。

**局所フェロモン更新**: 蟻がエッジを通過する度にフェロモンを減らすことで、同じ経路への過度の集中を防ぎ、多様な探索を促す。

```
τ_ij ← (1-ξ) * τ_ij + ξ * τ_0
```

### 3.2 MAX-MIN Ant System (MMAS)

**提案**: Stutzle & Hoos (2000)

フェロモン値に上下限を設ける改良:

| 特徴 | 内容 |
|------|------|
| フェロモン境界 | τ_min ≤ τ_ij ≤ τ_max を強制 |
| 更新蟻の制限 | 最良蟻のみがフェロモンを更新 |
| 初期フェロモン | τ_max で初期化（初期から幅広い探索を促進） |
| 再初期化 | 停滞検出時にフェロモンを τ_max に再初期化 |

**なぜ上下限が有効か**:

- `τ_max`: フェロモンの過剰蓄積を防ぎ、早期収束（premature convergence）を回避
- `τ_min`: 任意のエッジが選択される最低確率を保証し、探索の多様性を維持

MMASは多くのベンチマーク問題でASやACSより優れた結果を示している (Stutzle & Hoos, 2000)。

### 3.3 Rank-Based Ant System (AS_rank)

**提案**: Bullnheimer, Hartl & Strauss (1999)

蟻をツアー距離でランク付けし、ランクに応じた重みでフェロモンを更新:

```
Δτ_ij = Σ_{r=1}^{w-1} (w-r) * Δτ_ij^r + w * Δτ_ij^{gb}
```

- 上位 w-1 匹の蟻がランクに応じたフェロモンを付加
- グローバルベスト蟻が最大の重み w でフェロモンを付加
- ランクが低い蟻の影響を段階的に小さくすることで、良い解への集中と探索のバランスを取る

### 3.4 バリエーション比較まとめ

| 特徴 | AS | ACS | MMAS | AS_rank |
|------|-----|------|------|---------|
| 遷移ルール | 確率的 | 擬似ランダム比例 | 確率的 | 確率的 |
| 更新蟻 | 全蟻 | 最良蟻のみ | 最良蟻のみ | 上位 w 匹 + GB |
| フェロモン境界 | なし | なし | [τ_min, τ_max] | なし |
| 局所更新 | なし | あり | なし | なし |
| 停滞対策 | なし | 局所更新で緩和 | 再初期化 | ランクで緩和 |
| 典型的な性能 | 基準線 | ASより優れる | 最も安定して優秀 | ASより優れる |

## 4. 収束性の分析

### 4.1 確率的収束

Stutzle & Dorigo (2002) は、ACOアルゴリズムが最適解に収束する条件を分析した。

**主な結果**:
- フェロモンの蒸発がある場合、ASは最適解への収束が**保証されない**（フェロモンが均一化する可能性）
- ACSおよびMMASでは、最良蟻のみがフェロモンを更新するため、十分な反復回数で最適解に収束する確率が1に近づく
- ただし「十分な反復回数」は問題サイズに対して指数的に増大しうる

### 4.2 探索と活用のトレードオフ

ACOの性能は**探索（exploration）**と**活用（exploitation）**のバランスに依存する:

```
探索寄り                                    活用寄り
←─────────────────────────────────────────────→
多様な解を試す                              最良解の周辺を集中探索
局所解を脱出しやすい                         収束が速い
収束が遅い                                  局所解に陥りやすい
```

各バリエーションのバランス制御メカニズム:

| バリエーション | 探索の促進 | 活用の促進 |
|-------------|-----------|-----------|
| AS | 全蟻更新で多様性維持 | （特になし） |
| ACS | 局所フェロモン更新 | q_0 による決定的選択 |
| MMAS | τ_min で最低探索確率を保証 | 最良蟻のみ更新 |

## 5. パラメータの影響と設計指針

### 5.1 各パラメータの役割

| パラメータ | 影響 | 小さい値 | 大きい値 |
|-----------|------|---------|---------|
| α (フェロモン重み) | フェロモンへの依存度 | ヒューリスティック主導（貪欲） | フェロモン主導（経験重視） |
| β (ヒューリスティック重み) | 距離情報への依存度 | 距離を無視 | 近い都市を強く選好 |
| ρ (蒸発率) | 過去の経験の忘却速度 | 過去の経験が長く残る | 新しい経験を重視 |
| n_ants (蟻の数) | 1イテレーションの探索幅 | 探索が狭い | 探索が広いが計算コスト増 |
| q_0 (ACSのみ) | 探索/活用の比率 | 探索寄り | 活用寄り |

### 5.2 一般的な推奨値

Dorigo & Stutzle (2004) が示す経験的なガイドライン:

| パラメータ | AS | ACS | MMAS |
|-----------|-----|------|------|
| α | 1.0 | 1.0 | 1.0 |
| β | 2.0〜5.0 | 2.0〜5.0 | 2.0〜5.0 |
| ρ | 0.5 | 0.1 | 0.02〜0.05 |
| n_ants | 都市数と同じ | 10 | 都市数と同じ |
| q_0 | - | 0.9 | - |

**重要な知見**:
- `α=0` にするとフェロモンが無視され、単なる貪欲法に退化する
- `β=0` にすると距離情報が無視され、フェロモンのみで決定する（初期はランダムに近い）
- `ρ` が大きすぎると過去の良い解が忘れられ、小さすぎると劣った解のフェロモンが残る
- `n_ants` は計算コストとのトレードオフで決定する（一般に10〜都市数程度）

## 6. 本リポジトリ実装との対応

本リポジトリの [`simulation/ant_colony_optimization/aco.py`](/simulation/ant_colony_optimization/aco.py) は **Ant System (AS)** を実装している。

### 対応関係

| アルゴリズム要素 | 本実装の対応 |
|---------------|------------|
| 遷移確率 | `[τ_ij]^α * [η_ij]^β` に基づく確率的選択 |
| フェロモン蒸発 | `(1-ρ) * τ_ij` による全エッジ蒸発 |
| フェロモン更新 | **全蟻**がツアーに沿って `Q/L_k` を付加 |
| ヒューリスティック | 距離の逆数 `1/d_ij` |

### 拡張の候補

本調査を踏まえ、以下の拡張が検討できる:

1. **ACS への拡張**: 擬似ランダム比例ルールと局所フェロモン更新の追加
2. **MMAS への拡張**: フェロモン上下限の導入と停滞時再初期化
3. **パラメータ感度実験**: α, β, ρ を変動させた性能比較実験

## 応用事例

> 詳細は [`docs/research/03_applications.md`](../../03_applications.md) を参照。

ACOが特に有効な問題の特徴:

- **組み合わせ最適化**: TSP、VRP、スケジューリング等のNP困難問題
- **動的環境**: ネットワークルーティング（AntNet）のようにリアルタイムで変化する問題
- **制約付き最適化**: 複数の制約条件を持つ問題（車両容量、時間窓等）

TSPにおけるACOの性能は、最適解からの誤差が小規模問題で1%以下、大規模問題でも数%程度であり、計算時間と解の品質のバランスに優れる (Dorigo & Stutzle, 2004)。

## なぜ機能するか（考察）

ACOが組み合わせ最適化で機能する本質的な理由:

1. **正のフィードバックによる良解の強化**: 短い経路にフェロモンが蓄積し、集団的に良い解を学習する
2. **負のフィードバックによる劣解の忘却**: 蒸発によって古い・劣った経路の影響が減衰する
3. **確率的探索による多様性**: 決定論的でなく確率的に選択するため、局所解からの脱出が可能
4. **分散型の並列探索**: 複数の蟻が同時に異なる解を構築し、解空間を効率的にカバー
5. **構成的ヒューリスティック**: ヒューリスティック情報（距離の逆数）が探索を有望な方向に誘導する

これらは自然界の蟻が持つ「探索と活用のバランス」「環境を介した間接的学習」「分散型の問題解決」というメカニズムをアルゴリズムとして再現したものである。
