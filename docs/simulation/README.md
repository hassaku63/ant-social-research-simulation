# テーマ2: シミュレーション・応用実装 ガイドライン

このドキュメントは **必要時参照** 用のリファレンスです。
シミュレーション・実装タスクを開始する際に参照してください。

## このテーマの目的

蟻・蜂の行動システムを応用できる実問題を検討し、コンセプト→設計→実装→検証のサイクルを通じて、
応用の有効性・限界・課題を明らかにします。

## 実装サイクル

各シミュレーション・実装は以下のサイクルを経て進めます:

```
コンセプト検討 → 計画 → 仕様策定 → 実装 → 検証 → 振り返り
```

## ディレクトリ構成

```
docs/simulation/
├── README.md               # このファイル（実装ガイドライン）
├── index.md                # 実装済み・進行中プロジェクト一覧
└── <project-name>/         # プロジェクトごとのサブディレクトリ
    ├── concept.md          # コンセプト・問題定義
    ├── spec.md             # 仕様書
    └── retrospective.md    # 振り返り

src/
└── <project-name>/         # 対応する実装コード
```

## 仕様書の書き方

各シミュレーション・実装の仕様書は次の構成で作成します:

```markdown
# [プロジェクト名] 仕様書

## 問題定義
（どのような問題を解決するか、なぜ蟻・蜂の行動システムを応用するのか）

## 採用するアルゴリズム・モデル
（参照する行動システム、参考にした研究）

## 要件
### 機能要件
（実現すべき機能）
### 非機能要件
（パフォーマンス・拡張性等）

## アーキテクチャ
（全体設計、主要コンポーネント）

## インターフェース定義
（入出力・API仕様）

## 評価基準
（実装が成功したと判定する基準）

## 参照する調査ドキュメント
（docs/research/ 配下の関連ドキュメントへのリンク）
```

## 品質基準

実装は以下の基準を満たすこと:

- [ ] 対応する仕様書 (`docs/simulation/<project>/spec.md`) が存在する
- [ ] コードから仕様書へのリンク（コメント等）が含まれている
- [ ] 評価基準を満たすことが検証されている
- [ ] 振り返り (`docs/simulation/<project>/retrospective.md`) が記録されている
- [ ] 採用したアルゴリズムの根拠となる研究が `docs/research/` に調査済みである

## 新しいシミュレーションを始めるときの手順

1. `docs/simulation/index.md` を確認し、既存プロジェクトとの関連を把握する
2. `docs/simulation/<project-name>/concept.md` にコンセプトを記述する
3. `docs/workflow/templates/simulation-task.md` を使って計画を立てる
4. `docs/simulation/<project-name>/spec.md` に仕様書を作成する
5. `src/<project-name>/` に実装コードを作成する
6. 検証後に `docs/simulation/<project-name>/retrospective.md` に振り返りを記録する
7. `docs/simulation/index.md` を更新する
